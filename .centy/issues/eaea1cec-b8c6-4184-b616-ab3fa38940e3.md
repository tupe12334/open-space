# Enable maximum pedantic Clippy lints

## Description

Configure Clippy with the most pedantic lint rules possible across the entire repository. This ensures the highest code quality standards, catches subtle bugs, enforces idiomatic Rust, and maintains consistency throughout the codebase.

## Acceptance Criteria

* [ ] Add a workspace-level `clippy.toml` or configure lints in `Cargo.toml`
* [ ] Enable `#![warn(clippy::pedantic)]` for all crates in the workspace
* [ ] Enable `#![warn(clippy::nursery)]` for early detection of newer lint patterns
* [ ] Enable `#![warn(clippy::cargo)]` for Cargo manifest best practices
* [ ] Enable `#![warn(clippy::restriction)]` selectively (review and pick applicable lints, as enabling all restriction lints conflicts with pedantic)
* [ ] Fix or explicitly allow (with justification) all existing warnings raised by the new lint configuration
* [ ] Ensure CI runs `cargo clippy -- -D warnings` so pedantic violations fail the build
* [ ] Document any intentionally suppressed lints with `#[allow(...)]` and a comment explaining why
