# Place main Mac display at center of stage grid

## Problem

Currently, screens are placed on the stage in the order they are returned by CoreGraphics (or virtual displays), with no special treatment for the main Mac display. This means the main display can end up at any position in the grid — left, right, top, etc. — depending on the OS-reported order.

## Desired Behavior

The main Mac display (`CGDisplay::main()`) should **always** be placed at the **center position** of the stage grid:

- **1 display** → trivially centered
- **2 displays** → main display is index 0 (left), or center the layout so main is visually centered
- **3 displays** (1 row × 3 cols) → main display is the middle one (index 1)
- **4 displays** (2 rows × 2 cols) → main display is bottom-left (index 2), or use 1+3 layout with main on top centered
- **5 displays** (2 rows: 3 top + 2 bottom, or 2 top + 3 bottom) → main display is bottom-middle (index 4 in a 2×3 grid)
- **6 displays** (2 rows × 3 cols) → main display is the middle-bottom (index 4)

## Implementation Notes

The relevant code is in `src/modules/stage/systems.rs`:

1. **Display collection** (lines 49–72): After collecting `screen_specs`, identify which entry corresponds to `CGDisplay::main()` (display ID from `CGDisplay::main().id`).
2. **Reorder** the `screen_specs` vec so the main display ends up at the center index of the grid before the layout loop.
3. **Layout loop** (lines 122–134): No changes needed if reordering is done before this point — the existing grid math already centers the layout around the origin.

### Center index calculation

For a grid with `n` displays and `cols` columns:
- Total rows = `ceil(n / cols)`
- Center column = `cols / 2` (integer division)
- Center row = `total_rows / 2`
- Center index = `center_row * cols + center_col`

For odd counts (1, 3, 5), the center is unambiguous. For even counts (2, 4, 6), pick the bottom-middle position.
