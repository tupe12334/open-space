# Explore OpenXR as an alternative to ar-drivers-rs for Windows support

## Problem

The project currently only works on macOS because `ar-drivers-rs` has a Windows compatibility issue with its HID communication layer (see [ar-drivers-rs#13](https://github.com/badicsalex/ar-drivers-rs/issues/13)).

## Proposal

Investigate using [OpenXR](https://www.khronos.org/openxr/) (via [bevy_oxr](https://github.com/awtterpip/bevy_oxr)) as an alternative approach to gain cross-platform support, including Windows.

Also look into [Monado](https://monado.freedesktop.org/) — an open-source OpenXR runtime that could serve as the runtime layer on Linux and potentially Windows, removing the need for vendor-specific runtimes.

## Research Findings

### XREAL OpenXR Support (confirmed)

- **XREAL is an official OpenXR conformant vendor**, listed on the [Khronos conformant products page](https://www.khronos.org/conformance/adopters/conformant-products/openxr) alongside Meta, Microsoft, Valve, HTC, etc.
- **XREAL Air 2, Air 2 Pro, and Air 2 Ultra** are listed as supporting OpenXR.
- XREAL committed to integrating **OpenXR 1.1** across their entire product line, including through Nebula OS and their SDK.
- **Monado** has added a driver for the XREAL Air 2 Ultra, confirming open-source runtime support as well.
- **Note:** The original XREAL Air (v1) may not be covered — conformant listings mention Air 2 and newer.

## Considerations

### Pros
- Cross-platform support (Windows, Linux, macOS) — no dependency on `ar-drivers-rs` and its HID layer
- Standard API that could support more headsets beyond XREAL
- OpenXR is an industry standard backed by Khronos Group
- Monado provides a fully open-source OpenXR runtime, avoiding dependency on proprietary vendor runtimes
- XREAL has a conformant OpenXR runtime — no need for third-party workarounds

### Cons
- Significant architectural change — the project currently reads IMU data directly from the glasses via `ar-drivers-rs`, while OpenXR works through a runtime
- XREAL glasses would need a compatible OpenXR runtime installed on Windows (e.g., via XREAL's Nebula software, Monado, or a third-party runtime)
- May lose some low-level control over device communication
- Original XREAL Air (v1) may not have OpenXR runtime support

## Remaining Questions
- Does the original XREAL Air (v1) have OpenXR runtime support, or only Air 2+?
- What features would we gain/lose compared to direct HID access?
- Can both approaches coexist (OpenXR on Windows, ar-drivers-rs on macOS)?
- What is the current state of `bevy_oxr` maturity and compatibility?
