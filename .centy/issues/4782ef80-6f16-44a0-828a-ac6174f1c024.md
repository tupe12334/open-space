# Explore OpenXR as an alternative to ar-drivers-rs for Windows support

## Problem

The project currently only works on macOS because `ar-drivers-rs` has a Windows compatibility issue with its HID communication layer (see [ar-drivers-rs#13](https://github.com/badicsalex/ar-drivers-rs/issues/13)).

## Proposal

Investigate using [OpenXR](https://www.khronos.org/openxr/) (via [bevy_oxr](https://github.com/awtterpip/bevy_oxr)) as an alternative approach to gain cross-platform support, including Windows.

## Considerations

### Pros
- Cross-platform support (Windows, Linux, macOS) — no dependency on `ar-drivers-rs` and its HID layer
- Standard API that could support more headsets beyond XREAL
- OpenXR is an industry standard backed by Khronos Group

### Cons
- Significant architectural change — the project currently reads IMU data directly from the glasses via `ar-drivers-rs`, while OpenXR works through a runtime
- XREAL glasses would need a compatible OpenXR runtime installed on Windows (e.g., via XREAL's Nebula software or a third-party runtime)
- May lose some low-level control over device communication

## Questions to Explore
- Does XREAL provide an OpenXR runtime for their glasses?
- What features would we gain/lose compared to direct HID access?
- Can both approaches coexist (OpenXR on Windows, ar-drivers-rs on macOS)?
- What is the current state of `bevy_oxr` maturity and compatibility?
