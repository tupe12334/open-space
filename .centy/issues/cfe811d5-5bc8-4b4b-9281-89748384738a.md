# App crashes on macOS 26 due to objc2-foundation NSScreen enumeration type mismatch

## Description

The app compiles and starts successfully, creating 6 virtual displays (1920x1080 @ 60Hz), but then panics during NSScreen enumeration.

## Error

````
thread 'main' panicked at objc2-foundation-0.2.2/src/generated/NSEnumerator.rs:7:1:
invalid message send to -[_TtGCs23_ContiguousArrayStorageCSo8NSScreen_$ countByEnumeratingWithState:objects:count:]: expected return to have type code 'q', but found 'Q'
````

## Root Cause

The `objc2-foundation` v0.2.2 dependency expects `countByEnumeratingWithState:objects:count:` to return a signed integer (`q`), but on macOS 26.2 (Darwin 25.2.0) it returns an unsigned integer (`Q`). This is a type encoding mismatch between the crate and the newer macOS SDK.

## Environment

* macOS 26.2 (Darwin 25.2.0)
* Apple M4, 32 GiB RAM
* Rust edition 2021
* `objc2-foundation` v0.2.2, `objc2` v0.5.2

## Suggested Fix

Update `objc2` and `objc2-foundation` dependencies to newer versions that handle the unsigned return type on recent macOS releases.
